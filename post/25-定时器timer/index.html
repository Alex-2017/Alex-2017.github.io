定时器Timer

@BaseC · May 21, 2020 · 5 min read

1.Timer的介绍及使用

  Timer的主要任务是设置计划任务,但封装任务的类确是TimerTask类

1.1设置一个5s后执行的任务

MyTask.java

public class MyTask extends TimerTask {
    @Override
    public void run() {
        System.out.println("task begin run at " + new Date());
    }
}

TimerTest.java

public class TimerTest {
    public static void main(String[] args) {
        MyTask task = new MyTask();

        Calendar calendar = Calendar.getInstance();
        calendar.add(Calendar.SECOND, 5);

        //isDaemon 默认false,设置为true之后会将timer设置为守护线程,main线程结束运行之后,timer自动结束运行
        Timer timer = new Timer(false);
        System.out.println("start schedule at " + new Date());
        //任务将在5s后开始执行
        timer.schedule(task, calendar.getTime());
    }
}

控制台输出:

start schedule at Sun May 17 15:58:50 CST 2020
task begin run at Sun May 17 15:58:55 CST 2020

1.2Timer队列形式执行任务

  当Timer中有多个TimerTask时,TimerTask是以队列的方式一个一个被顺序的执行.当任务执行时间过长导致后面任务执行时间被延后时,后面的任务会在前面的任务执行完成之后直接执行.

MyDelayTask.java

public class MyDelayTask extends TimerTask {

    @Override
    public void run() {
        System.out.println("task begin run at " + new Date());
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println("task end run at " + new Date());
    }

}

TimerQueueTest.java

public class TimerQueueTest {

    public static void main(String[] args) {
        MyDelayTask task1 = new MyDelayTask();
        MyDelayTask task2 = new MyDelayTask();

        Calendar calendar1 = Calendar.getInstance();
        calendar1.add(Calendar.SECOND, 5);

        Calendar calendar2 = Calendar.getInstance();
        calendar2.add(Calendar.SECOND, 8);

        Timer timer = new Timer();
        System.out.println("start schedule at " + new Date());
        //延后性:多个TimerTask运行,若task运行时间晚于后面任务将要执行的时间,待正在执行的任务结束之后将会立即执行之后等待的任务
        //TimerTask执行任务的方式是队列的形式,FIFO;
        timer.schedule(task1, calendar1.getTime());
        timer.schedule(task2, calendar2.getTime());
    }

}

控制台输出:

start schedule at Sun May 17 15:59:44 CST 2020
task begin run at Sun May 17 15:59:49 CST 2020
task end run at Sun May 17 15:59:54 CST 2020
task begin run at Sun May 17 15:59:54 CST 2020
task end run at Sun May 17 15:59:59 CST 2020

1.3周期执行

TimerDelayTest.java

public class TimerDelayTest {

    public static void main(String[] args) {
        MyTask task = new MyTask();
        Timer timer = new Timer();
        System.out.println("start schedule at " + new Date());
        //延迟2s之后,每3秒执行一次
        //timer.schedule(task,2000,3000);

        Calendar calendar = Calendar.getInstance();

1.4TimerTask的cancel()方法

  TimerTask类中的cancel()方法的作用是将自身从任务队列中进行清除.

MyCancelTask.java

public class MyCancelTask extends TimerTask {

    @Override
    public void run() {
        System.out.println("cancel task begin run at " + new Date());
        //取消此次任务
        this.cancel();
        System.out.println("cancel task had been canceled");
    }

}

TimerTaskCancelTest.java

public class TimerTaskCancelTest {

    public static void main(String[] args) {
        MyCancelTask cancelTask = new MyCancelTask();
        MyTask myTask = new MyTask();

        Timer timer = new Timer();
        timer.schedule(cancelTask, new Date());
        timer.schedule(myTask, new Date());
    }

}

控制台输出:

cancel task begin run at Sun May 17 16:28:35 CST 2020
cancel task had been canceled
task begin run at Sun May 17 16:28:35 CST 2020

1.5Timer的cancel()方法

  Timer类中的cancel()作用是将任务队列中全部的任务进行清空.

TimerCancelTest.java

public class TimerCancelTest {

    public static void main(String[] args) {
        MyTask task1 = new MyTask();
        MyTask task2 = new MyTask();

        Timer timer = new Timer();
        timer.schedule(task1, new Date());
        timer.schedule(task2, new Date());

        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        timer.cancel();
    }

}

控制台输出:

task begin run at Sun May 17 16:29:27 CST 2020
task begin run at Sun May 17 16:29:27 CST 2020

1.5.1Timer类的cancel()方法注意事项

  Timer类的cancel()方法有时并不一定停止计划任务,而是正常执行.

MyTimerCancelExceTask.java

public class MyTimerCancelExceTask extends TimerTask {

    private int i;

    public MyTimerCancelExceTask(int i) {
        super();
        this.i = i;
    }

    @Override
    public void run() {
        System.out.println("第" + i + "次任务没有被cancel");
    }
}

TimerCancelExceTest.java

public class TimerCancelExceTest {

    public static void main(String[] args) {
        int i = 1;
        while (true) {
            MyTimerCancelExceTask task = new MyTimerCancelExceTask(i);
            i++;
            Timer timer = new Timer();
            timer.schedule(task, new Date());
            timer.cancel();
        }
    }

}

控制台输出:

第68612次任务没有被cancel

1.6延时与不延时不同任务的执行特点

1.6.1不延时任务

TimerDelayTest2.java

public class TimerDelayTest2 {

    public static void main(String[] args) {
        MyDelayTask delayTask = new MyDelayTask();
        Timer timer = new Timer();
        timer.schedule(delayTask, new Date(), 6000);
    }

}

控制台输出:

task begin run at Sun May 17 16:34:01 CST 2020
task end run at Sun May 17 16:34:06 CST 2020
task begin run at Sun May 17 16:34:07 CST 2020
task end run at Sun May 17 16:34:12 CST 2020
task begin run at Sun May 17 16:34:13 CST 2020
task end run at Sun May 17 16:34:18 CST 2020

**总结:**执行任务的时间是5s,小于period时间6s,下次执行任务的开始时间是上一次执行任务的开始时间加上period时间.

1.7追赶性

TimerCatchTest.java

public class TimerCatchTest {

    public static void main(String[] args) {
        MyTask task = new MyTask();

        Timer timer = new Timer();

        Calendar cal = Calendar.getInstance();
        cal.add(Calendar.SECOND, -20);

        //不具有追赶性,不会再执行已过去的时间段里的任务
        timer.schedule(task, cal.getTime(), 2000);
        //具有追赶性,会补偿执行已过去的时间段里的任务
        //timer.scheduleAtFixedRate(task, cal.getTime(), 2000);

    }

}

task begin run at Sun May 17 16:46:07 CST 2020
